# BIS Residential Property Prices — обзор данных

## 1. Что это такое?

BIS (Bank for International Settlements) — "банк центральных банков" в Базеле. Центробанки разных стран скидывают ему данные о ценах на жильё, а BIS приводит всё к единому формату и публикует. Бесплатно, без регистрации, без API-ключа.

Цель — дать возможность сравнивать рынки недвижимости разных стран в одном месте.

Обновляется раз в месяц (последняя неделя месяца). Данные выходят с лагом: квартальные цифры появляются примерно через 5 месяцев после окончания квартала.

**Лицензия:** данные доступны для некоммерческого использования с указанием источника. Коммерческое использование требует письменного разрешения BIS. Подробности: https://www.bis.org/terms_conditions.htm

## 2. Два датасета — SPP и DPP

### SPP (Selected Property Prices) — для сравнения стран

Один показатель на страну. BIS сам выбрал самый репрезентативный индикатор для каждой из 61 стран. Все приведены к одной шкале (2010 = 100), все квартальные. Идеально для графика "как росли цены в разных странах".

Для каждой страны — до 4 серий:

|  | Index (уровень цен) | YoY% (изменение за год) |
|--|---------------------|------------------------|
| **Nominal** (цены как есть) | Сколько стоит относительно 2010 | На сколько % выросло за год |
| **Real** (без инфляции) | То же, но инфляция вычтена | То же, инфляция вычтена |

Пример: US, 2020-Q1, Nominal, Index = 208 — значит цены выросли на 108% по сравнению с 2010 годом.

35 388 строк, данные с 1927 года (US) по 2025-Q4. Все 61 страна имеют полный набор из 4 серий. Пропусков в OBS_VALUE нет — каждая строка содержит значение. OBS_STATUS всегда `A: Normal value`. Чистый датасет.

**Колонки CSV (WS_SPP_csv_flat.csv):**

| # | Колонка | Описание | Пример значения |
|---|---------|----------|-----------------|
| 0 | STRUCTURE | Тип записи | `dataflow` |
| 1 | STRUCTURE_ID | Идентификатор датасета | `BIS:WS_SPP(1.0): Selected residential property prices` |
| 2 | ACTION | Действие | `I` |
| 3 | FREQ | Частота | `Q: Quarterly` |
| 4 | REF_AREA | Страна | `US: United States` |
| 5 | VALUE | Тип значения (nominal/real) | `N: Nominal` или `R: Real` |
| 6 | UNIT_MEASURE | Метрика | `628: Index, 2010 = 100` или `771: Year-on-year changes, in per cent` |
| 7 | TIME_PERIOD | Период | `2020-Q1` |
| 8 | OBS_VALUE | Числовое значение | `208.2134` |
| 9 | UNIT_MULT | Множитель | `0: Units` |
| 10 | BREAKS | Разрывы в серии | (обычно пусто) |
| 11 | COVERAGE | Покрытие | (обычно пусто) |
| 12 | TITLE_TS | Название серии | (обычно пусто) |
| 13 | OBS_STATUS | Статус наблюдения | `A: Normal value` |
| 14 | OBS_CONF | Конфиденциальность | `F: Free` |
| 15 | OBS_PRE_BREAK | Значение до разрыва | (обычно пусто) |

Важно: колонка VALUE (индекс 5) — это **не числовое значение**, а тип данных (Nominal/Real). Числовое значение в OBS_VALUE (индекс 8). Название запутывает.

### DPP (Detailed Property Prices) — детальная разбивка

Тут каждая страна отдаёт всё что у неё есть. Много серий на страну: отдельно квартиры, отдельно дома, отдельно столица, отдельно вся страна, отдельно новостройки.

Данные разнородные — у кого-то квартальные, у кого-то годовые или даже помесячные. Базовые годы индексов разные (у кого 2010=100, у кого 2015=100 — всего 72 варианта). Часть серий вообще в абсолютных ценах (евро, доллары), а не в индексах.

44 499 строк данных + 347 строк метаданных (без наблюдений). 347 уникальных серий (по dimension-ключу без FREQ; с FREQ — 349, потому что 2 серии имеют данные в двух частотах). Данные с 1819 года (Норвегия) по 2026-Q1.

**Колонки CSV (WS_DPP_csv_flat.csv):**

Dimensions (фильтры серии):

| # | Колонка | Описание | Пример |
|---|---------|----------|--------|
| 0-2 | STRUCTURE, STRUCTURE_ID, ACTION | Служебные | `dataflow`, `BIS:WS_DPP(1.0)`, `I` |
| 3 | FREQ | Частота | `Q: Quarterly`, `A: Annual`, `M: Monthly`, `H: Half-yearly` |
| 4 | REF_AREA | Страна | `AU: Australia` |
| 5 | COVERED_AREA | Где в стране | `0: Whole country`, `2: Capital city` |
| 6 | RE_TYPE | Тип жилья | `8: Flats`, `3: Single-family houses - detached` |
| 7 | RE_VINTAGE | Возраст | `0: All`, `1: Existing`, `2: New` |
| 8 | COMPILING_ORG | Кто собирал | `0: Central bank`, `2: Private sector` |
| 9 | PRICED_UNIT | Как считали цену | `0: Per dwelling`, `6: Pure price` |
| 10 | ADJUST_CODED | Сезонная корректировка | `0: Non seasonally adjusted` |

Данные наблюдения:

| # | Колонка | Описание |
|---|---------|----------|
| 11 | TIME_PERIOD | Период (4 формата, см. ниже) |
| 12 | OBS_VALUE | Числовое значение |

**4 формата TIME_PERIOD в DPP:**

| Формат | Пример | Количество строк | Частота |
|--------|--------|-------------------|---------|
| `YYYY-QN` | `2020-Q1` | 24 864 | Quarterly |
| `YYYY-MM` | `2020-01` | 18 381 | Monthly |
| `YYYY` | `2020` | 1 156 | Annual |
| `YYYY-SN` | `2000-S1` | 98 | Half-yearly (не YYYY-HN!) |

В SPP только один формат: `YYYY-QN`. Парсер DPP должен обрабатывать все 4 паттерна.

| # | Колонка | Описание |

| 20 | DECIMALS | Знаков после запятой |
| 21 | DOC_METHOD | Ссылка на документацию методологии |
| 22 | MEASURE_DETAIL | Тип единицы измерения |
| 23 | META_UPDATE | Дата обновления метаданных |
| 24 | PUBLICATIONS | Ссылки на публикации |
| 25 | TITLE_GRP | Заголовок серии |

| 27 | TITLE_GRP_NAT | Заголовок на национальном языке |
| 28 | UNIT_MEASURE | Единица измерения (`628: Index, 2010 = 100`, `EUR: Euro`...) |
| 29 | UNIT_MULT | Множитель |
| 30 | OBS_CONF | Конфиденциальность |
| 31 | OBS_PRE_BREAK | Значение до разрыва |
| 32 | OBS_STATUS | Статус наблюдения |

Ключевое отличие от SPP: DPP CSV гораздо шире — 33 колонки vs 16 у SPP. Колонки 13-29 содержат подробные метаданные о методологии сбора данных каждой серии (кто собирал, как считали, какие источники).

**ВАЖНО: UNIT_MEASURE (колонка 28) пустой во всех 44 499 строках данных.** Базовый год индекса или валюта хранится только в 347 строках метаданных (строки без TIME_PERIOD и OBS_VALUE). Чтобы узнать unit_measure серии, нужно JOIN по dimension-ключу **без FREQ** (country + covered_area + re_type + vintage + compiling_org + priced_unit + adjust). В метаданных FREQ всегда пустой. Проверено: все 347 серий данных имеют ровно 347 строк метаданных — 1:1 соответствие.

Это архитектурно важно для импорта: метаданные нельзя просто пропустить — из них нужно извлечь UNIT_MEASURE, TITLE_GRP, COVERAGE и другую информацию о серии.

## 3. Кому полезно?

- **Инвесторы** — сравнить рынки разных стран, найти где растёт, где падает, где пузырь. Например: смотришь SPP Real Index по 10 странам за 5 лет — видишь что Турция выросла на 200% а Япония стоит на месте. Понятно куда деньги текут и где риски.
- **Аналитики** — недвижимость как индикатор экономики. Пример: берёшь SPP Real Index по Германии — если индекс >130 при стагнирующих зарплатах, это сигнал перегрева рынка. YoY% показывает скорость: +15% в год — пузырь надувается, -5% — уже сдувается.
- **Центробанки** — мониторинг финансовой стабильности. DPP позволяет копнуть глубже: может вся страна в норме, но столица перегрета (covered_area = Capital vs Whole country).
- **Data-провайдеры** (как EODHD) — расширение покрытия для клиентов. BIS даёт готовые, очищенные данные в единообразном формате — остаётся обернуть в API.
- **Исследователи** — долгосрочные тренды. Норвегия с 1819 года — можно изучать циклы недвижимости за 200 лет, включая мировые войны и кризисы.

## 4. Dimensions — что это за фильтры

### В SPP — 4 параметра

| Что | Вопрос | Варианты |
|-----|--------|----------|
| **country** (REF_AREA) | Какая страна? | US, DE, GB... + агрегаты (XM = Еврозона, XW = Мир, 4T = Развивающиеся, 5R = Развитые) |
| **value_type** (VALUE) | С инфляцией или без? | N = Nominal (с инфляцией), R = Real (без) |
| **unit_measure** (UNIT_MEASURE) | Индекс или процент? | 628 = Index (2010=100), 771 = YoY% |
| **freq** (FREQ) | Как часто? | Всегда Q (квартал) |

**Nominal vs Real простыми словами:** квартира стоила 100к в 2010, стоит 200к в 2024. Nominal скажет "рост в 2 раза". Но инфляция за это время 40% — значит 100к тогда это как 140к сейчас. Real учтёт это и скажет "реальный рост в 1.4 раза". Nominal — цена на ценнике, Real — реальная покупательная способность.

**Index vs YoY% простыми словами:** Index = "где мы сейчас" (208 = на 108% выше 2010). YoY% = "куда движемся" (+5% = за последний год подорожало на 5%).

### В DPP — 8 параметров

Те же country и freq, плюс:

| Что | Вопрос | Примеры |
|-----|--------|---------|
| **covered_area** (COVERED_AREA) | Где в стране? | Вся страна / столица / крупные города / малые города / село |
| **property_type** (RE_TYPE) | Какое жильё? | Все типы / дома / квартиры / земля |
| **vintage** (RE_VINTAGE) | Новое или б/у? | Все / новостройки / вторичка |
| **compiling_org** (COMPILING_ORG) | Кто собирал данные? | Центробанк / статагентство / частная компания |
| **priced_unit** (PRICED_UNIT) | Как считали цену? | За объект / за кв.м. / чистая цена |
| **adjust** (ADJUST_CODED) | Сезонность убрали? | Да / нет |

Пример серии: "Австралия → Сидней → отдельные дома → все (новые+старые) → от CoreLogic → за объект → без сезонной корректировки → квартальные". Меняешь любой фильтр — получаешь другую серию.

## 5. Особенности и подводные камни

**Формат CSV:** каждая ячейка содержит и код, и название через двоеточие. Например `US: United States`, `Q: Quarterly`. При импорте нужно парсить — отделять код от названия.

**Путаница в колонке VALUE (SPP):** колонка называется `VALUE:Value`, но содержит не числовое значение, а тип данных — `N: Nominal` или `R: Real`. Числовое значение лежит в `OBS_VALUE`. Если не обратить внимание — можно перепутать.

**Пропуски:** не у всех стран одинаковая глубина. US — с 1927, большинство — с 1970-х, некоторые — только с 2000-х. У одних стран в DPP десятки серий, у других — пара.

**Метаданные в DPP:** ~347 строк без наблюдений — это описания методологии конкретных серий. У них пустые TIME_PERIOD и OBS_VALUE, зато заполнены COVERAGE, DATA_COMP, DOC_METHOD, PUBLICATIONS — подробное описание как именно каждая страна собирает свои данные. При импорте наблюдений их нужно пропускать, но они полезны как источник мета-информации для API.

**Разные базовые годы в DPP:** 72 варианта (2010=100, 2015=100, 2000=100, 1912=100...). Сравнивать серии DPP между собой напрямую нельзя без пересчёта.

**Часть серий DPP — в валютах** (EUR, USD, GBP, HUF, NOK, RUB...), а не в индексах. Это абсолютные цены, не индексы.

**Агрегаты в SPP:** коды XM (Еврозона), XW (Мир), 4T (Развивающиеся), 5R (Развитые) — это не страны, а расчётные средние по группам стран.

**Обновления:** раз в месяц, последняя неделя. Ревизии старых данных минимальны (менее 2%).

**Лицензия:** BIS разрешает использование данных для некоммерческих целей с указанием источника. Коммерческое использование требует письменного разрешения BIS.
